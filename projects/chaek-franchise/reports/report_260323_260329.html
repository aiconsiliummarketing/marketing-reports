S8심층 인사이트 (5가지 핵심 발견)
1. 광고비 축소가 효율 개선으로 이어짐: 메타 광고비를 58% 줄였으나 결과당 비용은 31% 개선(185K→128K). 6주차 280만원은 과잉 집행 구간이었을 가능성. 주당 100~150만원이 현재 타겟 풀 대비 적정 구간으로 판단.
2. 메타 리드폼이 전환 핵심 채널: 누적 메타리드 31건(전체 56건의 55.4%). 리드폼 결과당 비용 약 72K원으로 랜딩 전환 대비 우수. 리드폼 소재 비중 확대 검토 필요.
3. 네이버SA 전환 부진 지속: 누적 1건(전체의 1.8%). 주당 2만원 수준 예산으로는 볼륨 자체가 부족. 전환 키워드가 확인되지 않아 비용 대비 효과 의문.
4. 쿠팡 소스 분리 필요성 재확인: 누적 쿠팡 제외 전환 5건. form_start 중 쿠팡 1,138건으로 전체의 22%를 차지. 쿠팡 팝업 혼입이 데이터를 왜곡하므로 지속 모니터링 필요.
5. 차액가맹금 CRM(카카오 알림톡) 전환 발생: 누적 2건. 비광고 채널에서 전환이 발생하기 시작. 기존 DB를 활용한 리마케팅 효과로 판단. 알림톡 캠페인 확대 고려.
S9채널별 심층 진단
메타 심층 진단
핵심 소재 성과: closed_leadform이 리드폼 전환의 핵심(4건). rights_traffic은 클릭 최다이나 전환 미발생. margin_leadform은 전환 0건 — 즉시 중단 또는 소재 교체 필요.

CTR 추이: 1주차 1.60% → 3주차 1.69% → 5주차 2.16% → 6주차 2.30% → 7주차 2.85%. 점진적 상승 추세. 광고비 축소로 고반응 타겟에 집중된 효과.

핵심 과제: ① margin_leadform 중단 검토 ② 주당 100~150만원 예산에서 전환 최적화 ③ 새 소재 테스트 (사례 중심 크리에이티브)
네이버SA 심층 진단
채널 현황: 주당 2만원 수준 소규모 집행. 누적 전환 1건(5주차). 7주차 전환 0건.

비용 누수: 검색어별 시트에서 "쿠팡", "SKT", "개인정보유출" 등 비관련 검색어 유입 확인 필요. 확장 매칭으로 비관련 키워드 유입 시 제외 키워드 추가 필요.

핵심 과제: ① 비관련 검색어 제외 키워드 추가 ② 전환 키워드 식별 후 해당 키워드 입찰가 상향 ③ 예산 규모 자체가 작아 전환 기대 어려움 — 예산 증액 또는 채널 재검토
당근 심층 진단
현황: 3/20부터 광고 중단. 6주차 이후 데이터 없음.
중단 사유: 전환 0건(누적), CPC 대비 성과 부재. 타겟 특성상 프랜차이즈 가맹점주 도달에 한계.
향후: 별도 재개 계획 없음. 예산은 메타에 집중 배분.
S10향후 전략 제언
전략 방향: 메타 리드폼 중심 전환 확보 체제를 유지하되, 주당 100~150만원 예산 구간에서 효율 극대화. 네이버SA는 소규모 유지하면서 전환 키워드 발굴에 집중. 비광고 채널(CRM 알림톡)을 통한 전환 보완.

P1 즉시

margin_leadform 소재 중단: 7주차 전환 0건. 해당 예산(약 17만원/주)을 closed_leadform으로 재배분. 근거: 결과당 비용 무한대 vs closed_leadform 71,938원.

네이버SA 비관련 검색어 제외: 검색어 리포트 확인 후 "쿠팡", "SKT", "개인정보" 등 비관련 검색어 제외 키워드 등록. 비용 누수 차단.

GA 전환 데이터 정합성 점검: 집단소송플랫폼과 테스트페이지 간 중복 전환 여부 확인. 쿠팡 소스 필터 정상 작동 확인.

P2 1~2주 내

메타 새 소재 테스트: 실제 사례 기반 크리에이티브(예: "A브랜드 가맹점주, 차액가맹금 3,200만원 반환 확정") A/B 테스트. 현재 소재 피로도 누적 가능성.

카카오 알림톡 캠페인 확대: CRM 전환 2건 발생 확인. 미제출 대상자 리마인드 + 신규 DB 대상 초기 안내 알림톡 추가 발송.

테스트 페이지(0306) 트래픽 확대: 개인정보동의 단계 추가 폼의 전환율 검증을 위해 메타 트래픽 일부 배분.

P3 중기

네이버SA 예산 구조 재검토: 현재 주 2만원으로는 유의미한 데이터 수집 불가. 주 10만원으로 증액하여 2주간 전환 테스트 후 존폐 결정.

리드폼 vs 랜딩 전환 구조 최적화: 메타 리드폼 비중을 70%까지 확대하되, 랜딩 페이지 전환 퍼널(form_start→최종제출) 이탈 구간 개선 병행.

쿠팡 소스 완전 분리 체계 구축: GA 필터 + 보고서 자동화에서 쿠팡 소스를 100% 분리. 향후 쿠팡 전용 대시보드 별도 운영 검토.
